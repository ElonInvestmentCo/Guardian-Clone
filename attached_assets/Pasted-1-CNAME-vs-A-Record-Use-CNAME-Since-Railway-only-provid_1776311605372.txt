1. CNAME vs A Record
Use CNAME. Since Railway only provides guardian-trading-api-production.up.railway.app (not an IP), you must use CNAME. This is the correct approach.

2 & 3. EXACT DNS Records to ADD in OpenSRS
Type	Host	Value	TTL
CNAME	www	guardian-trading-api-production.up.railway.app	3600
That's it. One record.

4. Records to DELETE
Delete ALL of these from OpenSRS (if they exist):

Any A records on @ (root domain)
Any A records on www
Any CNAME records on @ (root domain) — OpenSRS doesn't allow this anyway
Any old/test records pointing to other services
Any records with typos or incorrect values
Important: Root Domain Limitation
⚠️ You cannot use CNAME on the root domain (guardiiantrading.com without www). OpenSRS doesn't support it.

This means:

www.guardiiantrading.com → Will work with the CNAME above
guardiiantrading.com (root) → Will NOT work with CNAME
Solution for root domain: If you need the root domain to work, you have two options:

Option A: Use URL redirect (Recommended)

Keep CNAME on www
Add a redirect rule in OpenSRS or your app to forward guardiiantrading.com → www.guardiiantrading.com
Option B: Use ANAME/ALIAS record (if OpenSRS supports it)

Some registrars offer ANAME/ALIAS records that work like CNAME on root
Check if OpenSRS has this feature
