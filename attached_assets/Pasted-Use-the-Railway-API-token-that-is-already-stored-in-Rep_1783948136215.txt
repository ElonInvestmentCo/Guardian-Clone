Your objective is to automatically synchronize environment variables between Railway and Replit without changing the application's behavior.

1. Connect to the Railway project using the existing Railway API token.

2. Inspect every Railway service and environment.

3. Retrieve all environment variables currently configured in Railway.

4. Compare them against the current Replit Secrets.

5. Identify any variables that exist in Railway but are missing from Replit Secrets.

6. Identify any variables that exist in Replit Secrets but are missing from Railway.

7. Synchronize the missing variables in both directions where appropriate so that Railway and Replit contain the same required configuration.

8. Do NOT overwrite existing values unless they are identical or confirmation can be made that the replacement is safe.

9. Do NOT rotate, regenerate, delete, rename, or modify any existing secret values.

10. Do NOT change any application code.

11. Do NOT modify build commands, deployment settings, workflows, services, databases, or infrastructure.

12. Do NOT alter any Cloudflare, Railway, PostgreSQL, OpenAI, Resend, authentication, or API configuration.

13. After synchronization, verify that:
    - every required environment variable exists,
    - every service can read its required variables,
    - there are no missing secrets,
    - no fallback or test credentials are being used.

14. Validate the application configuration without changing its behavior.

15. If any variable cannot be synchronized automatically due to Railway or Replit permission limitations, report exactly which variable is affected and why.

When finished, provide a report containing:

- Every environment variable discovered.
- Which variables exist in Railway.
- Which variables exist in Replit Secrets.
- Which variables were synchronized.
- Which variables could not be synchronized.
- Confirmation that no existing secrets or application configuration were modified.
