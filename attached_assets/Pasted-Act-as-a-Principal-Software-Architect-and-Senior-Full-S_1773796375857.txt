Act as a Principal Software Architect and Senior Full-Stack Engineer specializing in analytics platforms, ad attribution infrastructure, and high-performance event tracking systems.

Your task is to design and implement a **complete SaaS analytics and advertising attribution platform** similar to Google Analytics, Mixpanel, Amplitude, and Hotjar.

The system must allow **any website to install a small script and send analytics data to the platform**, where it will be processed and visualized in dashboards.

The platform must support:

• multi-tenant SaaS architecture
• event tracking
• campaign attribution
• session tracking
• real-time analytics
• heatmaps
• session replay
• AI-powered insights
• marketing attribution
• privacy-safe visitor identification

The system must be **fast, scalable, and optimized for millions of events**.

---

SYSTEM ARCHITECTURE

Design the system with these components:

1. CLIENT TRACKING SCRIPT
2. EVENT INGESTION API
3. EVENT QUEUE PIPELINE
4. ANALYTICS DATABASE
5. SESSION RECORDING ENGINE
6. HEATMAP GENERATION ENGINE
7. AI INSIGHTS ENGINE
8. ANALYTICS DASHBOARD
9. PROJECT MANAGEMENT SYSTEM

PART 1 — MULTI-TENANT PROJECT SYSTEM

Create a multi-tenant architecture where each user can create multiple projects (websites).

users

api_keys
events
visitors
sessions

id (UUID)
name
domain

id

PART 2 — EMBEDDABLE TRACKING SCRIPT

Create a lightweight JavaScript file:

https://yourdomain.com/tracking.js

Usage:

<script async src="https://yourdomain.com/tracking.js" data-key="PUBLIC_API_KEY"></script>

Tracking.js must:

• load asynchronously
• be under 10KB
• never block page rendering

Automatically track:

PageView
Clicks
Scroll depth
Form submissions
Custom events

Expose global API:

analytics.track()

Example:

analytics.track("signup_completed")

Automatically collect:

page_url
referrer
screen size
browser user agent
timezone
language
timestamp

Send events to:

POST /api/events

---

PART 3 — EVENT INGESTION API

Create endpoint:

POST /api/events

Payload example:

{
api_key: "PUBLIC_API_KEY",
event_type: "PageView",
event_name: "page_load",
page_url: "/pricing",
referrer: "google.com",
timestamp: 1712345678
}

Server must:

• validate API key
• identify project
• generate event_id
• send event to queue

---

PART 4 — EVENT PROCESSING PIPELINE

Use Redis with BullMQ.

Pipeline:

tracking.js → ingestion API → queue → worker → database

Workers process events asynchronously.
